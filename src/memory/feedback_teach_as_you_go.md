Always explain the "why" behind code as we write it — coding principles, architecture decisions, patterns used, tradeoffs made. Weave teaching into every step naturally alongside the code.

**Why:** User is learning and wants to understand the reasoning, not just receive code.

**How to apply:** After writing each file, explain key decisions: what pattern is being used, why that approach over alternatives, any relevant principle (e.g. separation of concerns, single responsibility, security boundaries).
